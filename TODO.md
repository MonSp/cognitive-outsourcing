# SIG 后续研究方向

> **核心叙事**: SIG 作为边缘小模型长任务流式注入专用加速器。
> SIG 不是通用缓存方案的替代品，而是资源受限边缘设备上，支撑 0.8B–4B 小模型执行超长时、多任务交织、状态持续累积代理任务的**不可替代的推理运行时原语**。

---

## 核心基准：EdgeAgent-Kitchen

### 设计目标

证明在资源受限的边缘设备（RTX 4070 SUPER / Jetson Orin 级）上，当 0.8B–4B 小模型需要执行超长时、多任务交织、状态持续累积的真实世界代理任务时，SIG 能提供**数量级层面的端到端效率与任务成功率**，而所有基于重编码（含前缀缓存）的基线将因累计延迟或内存溢出而不可行。

### 场景描述

一个智能厨房助手，在连续 3 小时内处理相互依赖的多类型任务，工具链长度达 50–200 步，包含：

1. **长期食谱规划**（30 步工具链）：根据用户口味、过敏原、现有食材生成一周菜单。
2. **实时烹饪指导**（20 步交错插入）：在烹饪过程中提供步骤提醒、替换建议。
3. **库存动态管理**（15 步触发式工具链）：当检测到食材短缺时，自动生成购物清单并比价。
4. **意外中断与重规划**（5 次随机注入）：如用户说"改做意大利菜"，需丢弃部分上下文并快速重排。

**关键特征**：
- 上下文累积超过 16K tokens，远超小模型单次编码的舒适区。
- 多个子任务共享部分历史上下文（用户档案、厨房状态），但又具有独立的分支状态。
- 频繁的小片段状态更新（如"烤箱预热到 180°C"）与大块工具结果注入（如食谱全文）交替。

### 基线设置

| 基线 | 描述 |
|------|------|
| **SIG (Stream)** | 核心基线，持久 KV 缓存，增量注入。 |
| **AppLoop-PC (Prefix Cached)** | 每次操作后利用前缀缓存重建完整上下文。 |
| **AppLoop-Sliding** | 上下文窗口滑动，超过长度则丢弃旧信息。 |
| **SIG-Hybrid** | 系统自动检测任务特性，对独立小片段使用 AppLoop-PC，长链使用 SIG（混合调度原型）。 |

### 评估指标

| 优先级 | 指标 | 说明 |
|--------|------|------|
| 🥇 主指标 | 端到端任务耗时（Wall-Clock Time） | 体现真实用户体验。 |
| 🥇 主指标 | 长链任务完成率 | 因内存不足或超时导致的失败次数。 |
| 🥇 主指标 | 上下文检索 F1 | 每隔 30 步插入"根据 50 步前的某条信息，现在该怎么做？"的探测问题，评估状态保持能力。 |
| 🥈 辅助 | 决策准确率（LLM-Judge） | 最终菜单、购物清单的合理性。 |
| 🥈 辅助 | 内存占用峰值 | 各基线在实际运行中的显存峰值。 |

### 预期结果（支撑核心叙事）

- 在 50+ 步交织任务中，SIG 的端到端耗时将是 AppLoop-PC 的 **5–10 倍优势**，且不会因上下文溢出而失败。
- AppLoop-PC 因前缀缓存无法覆盖多分支、交错的非共享状态，优势消失，退回为全量重编码。
- SIG-Hybrid 在维持高成功率的同时，可能在某些纯碎片环节取得额外微小增益，证明 SIG 不仅是一个算子，更应成为边缘推理运行时的核心调度原语。

## 研究方向：以 SIG 为中心的边缘代理设计空间

### R15: 混合推理调度——SIG 与 AppLoop-PC 的自适应切换

- **分支**: `R15-hybrid-scheduling`
- **Worktree**: `d:\trunk\SIG\output\cognitive-outsourcing-R15`
- **核心问题**: 能否根据任务图的结构特征（连续长度、上下文共享度、更新粒度）在线决定使用 SIG 还是前缀缓存，从而在所有条件下都接近最优？
- **研究内容**:
  - 实现轻量级决策器（基于简单启发式或小学习模型），根据任务特征实时选择 SIG 或 AppLoop-PC。
  - 基于 EdgeAgent-Kitchen 基准，比较纯 SIG、纯 AppLoop-PC 与混合策略在混合负载下的平均耗时和成功率。
  - 刻画决策器在不同任务特征分布下的 Pareto 前沿。
- **难度**: ⭐⭐⭐
- **状态**: 🔧 计划中
- **预期贡献**: 证明 SIG 不是要取代现有缓存系统，而是与其形成互补的混合推理架构——这是 SIG 走向实用系统的关键一步。

### R16: 多序列并发——SIG 对多任务持久状态的高效隔离

- **分支**: `R16-multi-sequence-concurrency`
- **Worktree**: `d:\trunk\SIG\output\cognitive-outsourcing-R16`
- **核心问题**: SIG 能否在单个推理实例中维持多个独立的持久 KV 序列，供不同任务或用户使用，且切换开销极低？
- **研究内容**:
  - 基于 llama.cpp 的多序列 API，同时运行 3 个独立的长工具链（如三个不同家庭的厨房规划）。
  - 测量序列切换的延迟、内存开销，以及与多个独立 AppLoop 实例的对比。
  - 探索 KV 缓存共享策略：不同序列间可共享的公共前缀（如系统提示）的复用。
- **难度**: ⭐⭐⭐
- **状态**: 🔧 计划中
- **预期贡献**: 展示 SIG 作为多租户边缘推理加速器的潜力，在智能家居、多机器人系统中极具价值。

### R17: 上下文老化与压缩——SIG 长程记忆的显式管理

- **分支**: `R17-context-aging-compression`
- **Worktree**: `d:\trunk\SIG\output\cognitive-outsourcing-R17`
- **核心问题**: 在超长链（>200 步）中，如何避免 KV 缓存无限增长导致速度下降和模型注意力弥散？能否在 SIG 框架内实现可逆的上下文压缩与选择性遗忘？
- **研究内容**:
  - 实施"重要性评分"机制，对旧工具结果进行摘要并替换原始 KV 片段，或直接丢弃低重要性 token。
  - 测量压缩前后的吞吐量、内存及关键信息检索 F1。
  - 探索可逆压缩方案：压缩后的 KV 片段能否在需要时近似恢复？
- **难度**: ⭐⭐⭐
- **状态**: 🔧 计划中
- **预期贡献**: 解决持久缓存最棘手的工程问题，使 SIG 支持无限长度的边缘会话成为可能。

### R18: 预填充与生成的流水线分离——SIG 对推测解码的天然适配

- **分支**: `R18-prefill-decode-pipeline`
- **Worktree**: `d:\trunk\SIG\output\cognitive-outsourcing-R18`
- **核心问题**: SIG 将工具结果注入视为"预填充"，后续生成为"解码"。这种分离是否能使小模型在注入时复用预填充阶段的计算，并与投机解码（Speculative Decoding）无缝结合？
- **研究内容**:
  - 在 llama.cpp 框架中，令 SIG 的注入操作不中断生成流水线，实现注入与生成的并行。
  - 对比引入投机解码后，SIG 与 AppLoop 的加速比变化。
  - 分析预填充-解码流水线在边缘 GPU 上的硬件利用率。
- **难度**: ⭐⭐⭐
- **状态**: 🔧 计划中
- **预期贡献**: 揭示 SIG 在推理系统优化层面的深层潜力，将其从"算法技巧"升格为一种与硬件特性契合的并行范式。

### R19: 边缘集群中的 SIG 片段路由——面向分布式代理

- **分支**: `R19-edge-cluster-fragment-routing`
- **Worktree**: `d:\trunk\SIG\output\cognitive-outsourcing-R19`
- **核心问题**: 当多个边缘设备共同执行一个任务（如家庭中多个摄像头+音响协同），如何将 SIG 的 KV 片段作为第一公民在网络中路由与合并？
- **研究内容**:
  - 在两台 Jetson Orin 上模拟分布式厨房场景，一台处理视觉食材识别，另一台处理食谱文本。
  - 设计轻量级片段序列化协议，通过 Wi-Fi 传输 KV 片段并合并到主控制器。
  - 测量端到端延迟与带宽占用，分析 KV 片段传输 vs 原始文本传输的通信效率。
- **难度**: ⭐⭐⭐
- **状态**: 🔧 计划中
- **预期贡献**: 将 SIG 从单机技术拓展为"边缘认知互联网"的交换格式，为去中心化的具身智能提供基础。

> **论文叙事线**: 这五个新方向不再是罗列独立的测量结果，而是以 SIG 为核心、持久流式注入为灵魂，系统性地探索边缘代理的混合调度、多租户并发、长程记忆管理、流水线并行与分布式路由——为新一代边缘推理系统提出一套完整的设计原则与参考架构。
