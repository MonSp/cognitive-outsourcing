## 摘要

现代大语言模型智能体框架（LangChain、AutoGPT、SWE-agent 等）普遍采用**无状态应用层循环**来协调工具调用——每次外部交互后，模型必须从头重编码全部上下文，导致二次方增长的预填充开销和注意力状态的灾难性断裂。本文综述 Cognitive Outsourcing（CO）与 Suspend-and-Inject Generation（SIG）研究计划的核心理念：将工具交互从应用层接口提升为**推理引擎级原语**，通过 KV-Cache 注入实现跨工具调用的认知连续性。

基于六篇系列论文的完整实证与理论体系，本文从三个层面展开综述：（1）**SIG 的核心理念**——KV-Cache 连续性原则、五阶段挂起-注入-恢复循环、以及"以存换算"的设计哲学；（2）**CO 的认知架构**——三层系统（意义编译器、注入引擎、认知模块生态）如何赋予轻量级边缘模型动态访问外部认知资源的能力；（3）**对现代智能体框架的启发**——有状态运行时、可组合加速正交性、Batch-SIG 的架构无关性、以及 KV-Cache 作为一等公民的统一优化框架（KFC）如何重塑智能体系统设计。

本文旨在为智能体推理框架的设计者提供一份从理念到实践的完整路线图，阐明 SIG 的优势区、设计边界、、、以及对下一代边缘-云协同智能体架构的深远影响。

**关键词**：认知外包，挂起-注入生成，KV-Cache 连续性，边缘智能体，推理引擎优化，智能体框架

## 1. 引言：智能体推理的范式困境

### 1.1 无状态循环的根本缺陷

当前主流 LLM 智能体框架采用统一的执行模式：模型生成一个工具调用 → 暂停生成 → 执行工具 → 将工具结果拼接到上下文 → **从头重编码整个对话历史** → 继续生成。这个"应用层循环"（AppLoop）存在三个根本缺陷：

1. **二次方预填充开销**。每次工具调用后，累积的上下文必须被完整重编码。对于一个 4B 模型在 30 步工具链上，预填充开销从第一步的几十毫秒增长到第三十步的数秒，总预填充时间超过 2 秒 [4]。
2. **注意力状态的灾难性断裂**。重编码意味着模型的 KV-Cache——其"工作记忆"——被丢弃并从零重建。对于跟踪空间关系、维护任务进度、保持推理轨迹的具身智能体而言，这是认知上的灾难 [1]。
3. **小模型的工具使用能力被摧毁**。实证数据显示，0.8B 模型在 AppLoop 模式下 6/9 个场景的工具准确率为 0%——反复重编码完全破坏了模型建立的任何上下文理解 [2]。

### 1.2 云端优化无法下沉到边缘

FlashAttention、RadixAttention、连续批处理等技术在云端服务系统（vLLM、TensorRT-LLM、SGLang）中已将预填充开销降至接近可忽略的水平。然而，这些优化**无法下沉到边缘设备**：

- llama.cpp（事实上的边缘推理库）不实现 FlashAttention；
- 嵌入式设备（Jetson、智能手机 NPU）缺乏所需 GPU 架构；
- 单用户、单实例的边缘部署不存在并发请求可供批处理分摊；
- 边缘工具调用（传感器、本地感知管道）延迟在 100ms 以内，预填充确实主导了端到端时间。

CO+SIG 明确定位在这一"正交前沿"：边缘原生的单实例智能体推理 [1]。

## 2. SIG 的核心理念

### 2.1 KV-Cache 连续性原则

SIG 的核心理念可以用一句话概括：**保留模型的工作记忆，而不是在每次工具调用后将其丢弃**。

标准自回归解码维护一个 KV-Cache，编码所有已处理 token 的注意力状态。这个缓存是模型的*工作记忆*——它对对话历史和当前推理轨迹的隐式理解。SIG 的创新在于：当模型需要外部信息时，不是丢弃这个缓存并从头重建，而是**将外部信息注入到现有缓存中**，使模型能够从被中断的确切位置恢复推理，同时获得了新信息。

这一理念的形式化表述是：在标准 AppLoop 下，工具调用后完整前缀 (X, G_{<t}, R) 被重新编码，产生隐藏状态 H^{full}；在 SIG 注入下，只有工具结果 token R 经过预填充，使用挂起的 KV-Cache 作为前缀，产生 H^{inj}。关键的实证发现是 [2]：

> **SIG 和 AppLoop 以几乎相同的每 token 速率生成文本（108 vs 103 tok/s，±5% 差异）。** 速度提升完全来自预填充的消除，而非生成路径的改变。

这一发现具有深刻含义：SIG 改变了生成开始时 KV-Cache 中的内容，但不改变自回归解码路径本身。它是一个**纯粹的预填充层优化**。

### 2.2 五阶段挂起-注入-恢复循环

SIG 通过一个精心设计的五阶段循环将工具交互从应用层接口提升为推理引擎原语 [1]：

1. **挂起（Suspend）**：当生成流中检测到预定义的挂起标记（如 <<<TOOL>>>）时，暂停自回归解码；整个 KV-Cache 被保留。
2. **解析（Resolve）**：解析标记后的文本，识别请求的认知模块及其参数。
3. **获取（Fetch）**：注入引擎调用指定模块——本地传感器、云端教师、技能库等。
4. **注入（Inject）**：模块的文本响应被分词，包裹在稳定化模板中，以挂起的 KV-Cache 为前缀执行一次前向传播。这将注入的 token 追加到缓存中，不重新计算任何先前上下文。
5. **恢复（Resume）**：自回归解码从扩展后的缓存继续，模型现在包含了新信息。

由于只有注入的 token（加上模板）经过预填充，成本与注入大小线性相关，与总对话长度无关。这与 AppLoop 的二次方重编码成本形成鲜明对比。

**稳定化模板**是这一机制的关键工程细节：小型语言模型（<1B 参数）对外部工具输出的分布偏移敏感。每个注入都被包裹在一个结构化前导中，将格式错误率从 >30% 降至 <2%，代价仅为每次注入约 15-20 个额外 token [1]。

### 2.3 "以存换算"的设计哲学

SIG 与 Mooncake（云端 KV-Cache 中心化分解服务架构，FAST 2025 最佳论文）独立地收敛到了同一个设计哲学 [6]：

> **以存换算（Trading Storage for Computation）**：扩展 KV-Cache 存储以减少 GPU 重计算，其收益大于存储成本。

这一哲学在不同部署体制下有不同表现：

| 维度 | 云端（Mooncake） | 边缘（SIG） |
|------|-----------------|-------------|
| 存储介质 | VRAM → DRAM → SSD（跨集群 TB 级） | VRAM → DRAM（单设备 GB 级） |
| 缓存复用范围 | 跨请求、跨节点（全局池） | 跨工具调用、单请求内（注入连续性） |
| 计算-存储权重 | γ ≫ α（存储主导） | α ≫ γ（预填充主导） |
| 调度复杂度 | 全局 Conductor | 无调度（直接执行） |

KV-Cache-as-First-Class-Citizen（KFC）框架 [6] 将两者形式化为同一优化目标的不同体制特化：

$$\min_{\mathcal{A}} \Phi(\mathcal{A}) = \sum_{r \in \mathcal{R}} \alpha \cdot C_{\text{prefill}}(r, \mathcal{A}) + \beta \cdot C_{\text{transfer}}(r, \mathcal{A}) + \gamma \cdot C_{\text{storage}}(r, \mathcal{A})$$

Mooncake 是云端约束下的最优解（多租户、集群规模、吞吐量导向）；SIG 是边缘约束下的最优解（单用户、设备规模、延迟导向）。两者是同一原则的对偶 [6]。

## 3. CO 的认知架构

### 3.1 三层系统架构

Cognitive Outsourcing 将边缘智能组织为三层 [1]：

**第一层：意义编译器（Meaning Compiler）**。完全在设备上运行的轻量级自回归语言模型（0.8B-4B 参数）。其职责范围狭窄：解析用户指令为结构化意图、决定需要哪些认知模块、发出带正确模块规范的 SIG 挂起标记、从所有注入的上下文中合成连贯响应。关键洞察是：意义编译器不需要存储世界知识或拥有高级推理技能；它只需要稳健的指令跟随和上下文内合成能力。

**第二层：注入引擎（Injection Engine）**。扩展设备推理库（如 llama.cpp）的薄运行时层。它拦截生成流、管理 KV-Cache、解析工具描述、协调模块调用、执行安全策略（注意力掩码、输出消毒、模块隔离）。

**第三层：认知模块生态系统（Cognitive Module Ecosystem）**。符合标准清单的文本服务，包括：
- **本地感知与动作模块**：物体检测、SLAM 查询、运动规划——完全在设备上运行，保护传感器隐私。
- **云端教师模块**：前沿 LLM（GPT-4、Claude）被包装为推理脚手架——零样本、上下文内的知识蒸馏。
- **本地认知缓存**：成功推理链的持久记忆，支持亚毫秒级复用——是 Robo-Cortex [5] 等持续学习智能体蒸馏的启发式原则的自然存储库。
- **技能库**：预编码的原子动作序列，可作为结构化提示注入。

### 3.2 云端教师与上下文内模仿

当意义编译器遇到超出其推理能力的任务时，它调用云端教师。教师的响应被包裹在模板中，指导本地模型总结和适应专家推理——一种零样本、上下文内的知识蒸馏 [1]。

实证测量显示 [2]：在 0.8B 模型上，自主 AppLoop 准确率为 0.20；添加 4B 教师的预计算 CoT 计划加 SIG 将其提升到 1.00——CoT 放大 +0.80（80 个百分点），SIG 放大 +0.59（59 个百分点）。教师质量边际（4B 单独减去 0.8B 单独）为 0.72。在 5× 师生容量比下，CoT 和 SIG 独立贡献了显著且部分重叠但互补的增益。

## 4. 实证体系：从优势区到设计边界

### 4.1 预填充节省的鲁棒性

SIG 的预填充节省在模型大小、操作模式和架构之间表现出高度鲁棒性 [2, 3]：

| 指标 | 0.8B | 4B |
|------|------|-----|
| 预填充 token 节省 | 73-93% | 78-97% |
| 端到端加速（教师预计算模式） | 2.38× | 2.70× |
| 深度链峰值加速 | 4.96× | 5.26× |
| 30 步工具链加速（N=30） | 2.79× | 4.26-4.38× |
| 50 步 Kitchen 基准加速 | — | 5.3× |
| GPU VRAM 开销 | +0.1 GB | +0.1 GB |

跨架构验证 [3] 揭示了关键的架构依赖性：

| 架构 | SIG 预填充加速 | Batch-SIG vs AppLoop-PC |
|------|---------------|------------------------|
| Qwen3.5（稠密注意力） | 2.38-2.70× ✅ | 4.65× ✅ |
| Gemma-4-E2B（GQA + SWA） | 1.12× ⚠️ | 6.82× ✅ |
| Nemotron-3-Nano-4B（混合 Mamba+注意力） | 0.98× ❌ | 4.24× ✅ |

**核心发现**：原始预填充节省是 Qwen 系列特有的，但 Batch-SIG 架构无关 [3]。Batch-SIG 通过在单次生成调用前累积工具结果，实现了 4.24-6.82× 的跨架构加速——这是整个研究计划中最鲁棒的跨架构发现。

### 4.2 KV-Cache 连续性对小模型的拯救效应

SIG 对小模型多轮工具使用能力的影响是戏剧性的 [2]：

在 0.8B 模型的自主模式下，AppLoop 在 6/9 个场景上灾难性失败（0% 工具准确率）——反复重编码完全摧毁了模型建立的任何上下文。SIG 拯救了其中 5 个场景，达到 68-100% 准确率。这提供了间接但强有力的证据：KV-Cache 连续性对小模型多轮智能体能力至关重要。

更深入的 KV-Cache 探测实验 [3] 精确化了这一发现的机制：
- 单实体补全：SIG 注入后 100% 成功——信息确实存在于 KV-Cache 中且注意力可达。
- 多实体枚举：SIG 和 AppLoop 均为 0/5——这是模型容量限制，非 SIG 特有缺陷。

这揭示了一个根本性区分：KV-Cache 注入保留了联想式访问（类比、约束满足、部分补全），但丢失了枚举式访问（穷举列表、精确格式化召回）[3]。

### 4.3 设计边界：何时不应使用 SIG

一个成熟的加速范式必须有明确定义的边界。SIG 的设计边界已被系统性地刻画 [3, 4]：

**碎片化上下文组装（R13）**。当工作负载由许多小的、不相关的片段组成时，SIG 比 AppLoop-PC 慢 3.0×（0.8B）。SIG 优化的是"注入一次，连续消费"，而非频繁的小规模重建。

**随机访问检索（R8）**。AppLoop-PC 实现 50-64% 检索命中率，而 SIG 在 0.8B 上为 0%。SIG 的持久缓存是流式磁带，不是随机访问数组。

**云端依赖工具（>300ms 延迟）**。SIG 的加速从 5.49×（0ms）崩溃到 1.08×（500ms）。当工具延迟主导时，预填充节省变得微不足道。

**非 Qwen 稠密注意力架构**。原始预填充节省不迁移到混合 Mamba+注意力（Nemotron，0.98×）或 GQA 架构（Gemma-4，1.12×）。

这些边界不是 SIG 的失败——它们是流式注入范式在设计范围之外的预期行为，正如磁带驱动器在无法匹配 RAM 的随机访问延迟时并非"失败" [3]。

## 5. 理论基础：五维分析框架

### 5.1 信息论基础（R1）

SIG 注入与完整重编码之间的注意力分布偏移已在 Qwen2.5-0.5B 上进行了首次直接测量 [2]：

| 层组 | 头一致率 | 余弦相似度 | 解释 |
|------|---------|-----------|------|
| 早期（0-7） | 0.252 | 0.647 | 最敏感 — 基础注意力模式最受扰动 |
| 中期（8-15） | 0.304 | 0.735 | 通过自注意力部分恢复 |
| 晚期（16-23） | 0.427 | 0.793 | 最稳定 — 任务细化层受影响最小 |

尽管头一致率仅为 0.25-0.43，每 token 生成速率和 Token-Jaccard 忠实度几乎相同。这一张力通过表示冗余的概念解决：Transformer 架构中多条注意力通路可以编码相同的语义内容——模型可能注意到不同位置，但通过分布式编码到达任务相关事实的相似内部表示 [2]。

### 5.2 KV-Cache 生命周期与退化分析（R2）

64 轮深度验证实验 [2, 3] 在三个模型家族上测量了多轮事实召回：

| 模型 | 最大轮数 | 最大缓存 Token | 长期召回 | 退化？ |
|------|---------|---------------|---------|--------|
| Qwen3.5-0.8B | 64 | 13,574 | 0.93（稳定） | 否 |
| Qwen3.5-4B | 32 | 6,800 | 0.93（稳定） | 否 |
| Gemma-4-E2B | 32 | 6,799 | 1.00（完美） | 否 |

在 64 轮注入和 13.6K 缓存 token 的范围内，未观察到可测量的退化。早期报告的 0.50 召回分数被证实为测量伪影（提示格式错误和思考模式干扰），而非真实退化 [2]。

### 5.3 正交加速框架（R5 Paper）

SIG 的宏观级预填充消除与推测解码的微观级生成压缩作用于推理管道的正交维度 [5]。正交性比率 ρ 的形式化定义为：

ρ = S_{SIG+MTP} / (S_{SIG} × S_{MTP})

当 ρ = 1.0 时，两种加速完美乘法组合。实证验证 [5]：

| 验证方式 | ρ | 判定 |
|---------|--------|------|
| n-gram 顺序验证（4B） | 0.851 | PASS（≥ 0.85） |
| 原生 MTP 并行验证（4B，Kitchen） | 1.239 | PASS，略超乘法性 |

复合 SIG+MTP 实现 4.52× 端到端加速（SIG 3.50× + MTP 1.27×），验证了正交加速框架的实践可行性。ρ > 1.0 的略超乘法性源于 SIG 的缓存持久性提升了 MTP 草稿接受率——一种正向交互效应。

## 6. 对现代智能体框架的启发

### 6.1 有状态运行时：从无状态循环到认知连续性

当前智能体框架（LangChain、LlamaIndex、AutoGPT）普遍采用无状态应用层循环。SIG 的核心启发是：智能体框架应该将 KV-Cache 管理作为一等公民，而不是依赖应用层的上下文拼接。

具体而言，这要求推理引擎暴露三个原语：
- suspend()：暂停生成，保留 KV-Cache；
- inject(tokens)：将外部信息注入现有缓存；
- resume()：从扩展后的缓存继续生成。

这将工具交互从"API 调用 + 上下文重建"转变为"缓存状态转换"，从根本上改变了智能体与外部世界的交互模型。对于持续学习智能体（如 Robo-Cortex [1]），这意味着：

- 连续反思：KV-Cache 保留任务执行的完整上下文；当智能体进入反思阶段时，可以注入历史摘要而不重置空间记忆。
- 启发式缓存：自主知识归纳产生的原则被存储为本地认知模块，遇到类似情况时直接注入，零额外推理成本。
- 从外包到内化：从向云端"外包"认知，到"缓存"成功模式，到逐步将其"内化"为本地、无参数的技能——边缘认知进化的路径。

### 6.2 可组合加速的正交性原则

SIG 研究计划揭示了一个更广泛的设计原则 [5]：针对推理管道不同阶段的加速可以乘法组合，只要它们作用于独立的计算维度。

这一原则的实践含义是深远的。现代智能体框架可以将多种优化策略系统性地组合：

| 加速维度 | 机制 | 目标阶段 | 与其他维度的关系 |
|---------|------|---------|----------------|
| SIG（宏观） | KV-Cache 注入 | 预填充 | 与 MTP 正交 |
| MTP（微观） | 多 token 预测 | 生成 | 与 SIG 正交 |
| Batch-SIG（结构） | 批量注入 | 生成调用 | 架构无关 |
| CompSIG（内存） | 周期性压缩 | 缓存管理 | 与上述正交 |

正交性框架为评估任何复合加速是否值得实现努力提供了定量工具：如果 ρ 预计低于 0.70，加速可能相互干扰，不应朴素组合 [5]。

### 6.3 边缘智能体的隐私保护范式

CO 架构的隐私保证源于一个结构性属性：敏感用户数据保留在边缘设备上，只有消毒后的查询被发送给云端教师 [1, 2]。

隐私泄露被形式化为三个通道 [2]：
- 直接泄露 L_{direct} = I(U; Q)：查询文本中明确包含的 PII。
- 间接泄露 L_{indirect} = I(U; φ(Q) | explicit(Q))：从查询结构、风格或模式可推断的信息。
- 工具参数泄露 L_{tool} = Σ_t I(U; args_t)：通过外部工具 API 参数暴露的信息。

概念演示 [2] 展示了两种机制：PII 编辑（正则表达式命名实体检测）和仅意图外包（将领域特定值抽象为类型化占位符）。虽然形式化差分隐私保证仍为未来工作，但 CO 的架构性隐私属性——边缘编排、云端辅助——为隐私敏感领域（医疗、金融、法律）的智能体部署提供了结构性基础。

### 6.4 KV-Cache 作为一等公民：KFC 统一框架

KFC 框架 [6] 将 SIG 和 Mooncake 统一为同一优化原则的体制特化，其对智能体框架设计的核心启发是：

> KV-Cache 不是计算的副产品，而是推理系统设计的中心资源。

这一理念的实践落地包括：

1. KV-Cache 持久化。将缓存前缀哈希和块存储在本地 SSD 上，后续会话无需重编码即可恢复——等效于 Mooncake 的前缀哈希匹配在边缘的本地化版本 [6]。工程成本约 2-3 个月。

2. 多层存储。CompSIG 的压缩管道扩展以 DRAM 卸载：VRAM 持有活跃缓存（~8,000 token），DRAM 持有压缩缓存（~20,000 token），有效上下文扩展 3.5× [6]。

3. Batch-SIG 作为默认部署策略。跨所有架构推荐 Batch-SIG 作为默认策略 [3]。通过在单次生成调用前累积工具结果，Batch-SIG 将 SIG 的最坏情况（碎片化组装）转化为优势区（4.24-6.82× vs AppLoop-PC）。自动批量大小选择通过 DependencyAnalyzer 实现，将工具调用分类为独立（可批处理）和顺序依赖（不可批处理）[3]。

### 6.5 混合调度与决策框架

SIG 研究计划产出的决策框架 [3, 4] 可直接作为边缘推理运行时的路由启发式：

| 工作负载 | 使用 | 理由 |
|---------|------|------|
| 长连续工具链 | SIG | 增量成本恒定；前缀缓存无效 |
| 碎片化独立组装 | AppLoop-PC 或 Batch-SIG | SIG 的逐步开销超过单次重编码 |
| 随机访问检索 | AppLoop-PC | 完整重编码提供直接位置访问 |
| CoT 结构化生成 | 任一 | 性能等效；SIG 增加增量扩展能力 |
| 云端依赖工具（>300ms） | 任一 | SIG 优势崩溃至 <1.15× |
| 安全关键（回滚） | SIG + 隔离 | 回滚实现关键词清洁恢复 |

边缘推理运行时可以在运行时检测工作负载特征并相应路由：深度链到 SIG，碎片化组装到 Batch-SIG，检索工作负载到 AppLoop-PC，云端任务到较低开销的方案。这类似于现代编译器根据代码特征在内联（流式）和函数调用（重建）之间选择 [3]。

## 7. 局限性与未来方向

### 7.1 已知局限性

1. 单模型家族。大部分核心基准测试在 Qwen3.5（0.8B/4B，Q4_K_M）上进行。跨架构验证已扩展到 Gemma-4 和 Nemotron，但 Llama、Phi 等架构仍待测试 [3]。
2. 质量-速度权衡。SIG 模式下的任务质量存在可测量但更窄的差距：4B 上混合 TF-IDF 评分为 −0.12（关键词评分为 −0.39）。差距集中在配方枚举上，而非工具执行或过敏原意识 [3]。
3. 工具延迟敏感性。SIG 的端到端加速在 500ms 工具延迟下崩溃至 1.08× [3]。这限定 SIG 的价值主张在本地工具执行快速（<100ms）的场景。
4. 缓存枚举限制。KV-Cache 注入支持单实体联想访问（100% 补全），但不支持多实体穷举枚举（0%）[3]。这是 SIG 的基本架构限制。
5. 合成工具。所有基准使用合成工具实现。真实世界工具的延迟、噪声和可变响应长度可能以未捕获的方式与 SIG 的注入粒度交互。

### 7.2 未来方向

1. 本地 KV-Cache 磁盘持久化（最高优先级）。在 llama.cpp 中实现磁盘支持的 KV-Cache 持久化，使跨会话前缀复用无需网络连接。工程成本约 2-3 个月，收益相当于完整混合架构但成本仅 1/6 [6]。
2. KV-Cache 架构补救。SIG 无法可靠地从分布式 KV-Cache 状态枚举实体的根本限制需要架构级解决方案，如为关键实体保留显式 token 槽位的混合 KV-Cache 布局 [3]。
3. 跨家族复制。在 Llama、Gemma 和 Mistral 的 ≥7B 规模上验证 R6 的深度链优势。
4. Batch-SIG 依赖分类器的运行时集成。将 DependencyAnalyzer 集成到边缘推理运行时，实现自动批量大小选择 [3]。
5. 具身智能体的物理机器人部署。当前基准将物理动作抽象为基于文本的工具调用。在物理机器人上部署 SIG+MTP 验证框架在真实条件下的表现。
6. KV-Cache 标准化。提议标准化 KV-Cache 元数据模式，解决 llama.cpp、vLLM、SGLang、TensorRT-LLM 之间的格式碎片化 [6]。

## 8. 结论

Suspend-and-Inject Generation 的核心理念可以凝练为三个原则：

1. **工具交互是推理引擎原语，不是应用层接口。** 通过将挂起-注入-恢复循环实现在推理引擎层，SIG 从根本上改变了智能体与外部世界的交互模型——从"API 调用 + 上文重建"到"缓存状态转换"。

2. **KV-Cache 是工作记忆，不是临时数据。** 保留模型的注意力状态跨越工具调用边界，不仅消除了 73-97% 的冗余预填充，更重要的是维持了认知连续性——对小模型的多轮工具使用能力具有决定性影响。

3. **以存换算：存储换取计算是跨体制的统一原则。** 从边缘的注入连续性到云端的全局 KV-Cache 池，KV-Cache 作为一等公民的管理原则是统一的，只是在不同部署体制下的表现形式不同。

对现代智能体框架而言，CO-SIG 研究计划的启发是结构性的：它不仅仅是另一个性能优化技巧，而是提出了一种重新思考智能体推理管道的范式——有状态的、以 KV-Cache 为中心的、可组合加速的、隐私保护的边缘智能体运行时。这一范式的价值随着模型规模增长和任务复杂度增加而增长，其设计边界已被系统性刻画，其核心机制已被跨架构验证。

正如 Mooncake 和 SIG 从相反的起点独立收敛到同一设计哲学所暗示的：随着模型变得更大、上下文窗口变得更长，KV-Cache 管理将成为推理系统设计的中心关切，无论部署体制如何。将 KV-Cache 视为一等公民的系统——保留它、共享它、高效移动它——将优于将其视为临时副产品的系统。
